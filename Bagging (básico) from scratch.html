Bagging from scratch

Bagging, tambi√©n conocido como bootstrap aggregating (agregaci√≥n de bootstrap) es un meta-algoritmo de machine learning que busca mejorar la precisi√≥n de predicciones de modelos de aprendizaje autom√°tico en tareas como la clasificaci√≥n y regresi√≥n. Si bien el m√°s conocido de los modelos de bagging es el random forest, este puede ser aplicado con cualquier tipo de sub-m√©todo o sub-modelo.Antes de empezar con el modelo y hacer c√≥digo, tenemos que entender un concepto base detr√°s de todo, el bootstrapping.

El bootstrapping es un m√©todo de remuestreo utilizado para aproximar distribuciones utilizando muestras con reemplazo de la muestra original, la imagen a continuaci√≥n resume un proceso simple de bootstrapping

Como se aprecia en la imagen anterior, disponemos de una muestra de 10 elementos, con tres elementos del color verde claro, tres elementos naranja, y cuatro elementos azules. Al hacer las muestras bootstrap notamos como se duplican elementos de la muestra original, la primera muestra bootstrap tiene dos veces el elemento 6 (naranjo), dos veces el elemento 5 (naranjo), dos veces los elementos 1 y 4 (verde claro), y dos veces el elemento 8 (azul). De esta forma aumentamos el tama√±o aplicando reemplazo de elementos desde el conjunto de datos original.

Entendiendo esto podemos introducir el concepto de bagging. Supongamos que tenemos un modelo que realiza alguna tarea de clasificaci√≥n o regresi√≥n, si el modelo es bueno, ¬ømejorar√° el desempe√±o tener muchos m√°s modelos trabajando en conjunto? Esta idea se muestra en la imagen a continuaci√≥n

Ahora vamos a construir nuestro modelo y comparar respecto a los modelos base, para esto primero se ajusta un arbol de decisi√≥n (bien b√°sico sin ajuste de par√°metros ni nada), y luego se compara ese modelo b√°sico en un bagging.

Para realizar el bagging generamos una funci√≥n que permite crear una cantidad definidad por el usuario de modelos (argumento N_arboles). La funci√≥n tiene la siguiente estructura.

  • Crea un objeto lista vac√≠o que llenar√° con los arboles creados en un loop.
  • Extra los √≠ndices desde el conjunto de entrenamiento con los que har√° el muestreo bootstrap
  • En cada iteraci√≥n crea un √°rbol de decisi√≥n que entrena con muestras bootstrap del conjunto de entrenamiento (X e Y).
  • Junta todos estos objetos en la lista generada al principio y devuelve ese elemento desde la funci√≥n

def create_random_forest(N_arboles):
    
    objs = list() # Lista vac√≠a para guardar los modelos
    indices = X_train.index # √≠ndices para muestrear

    # Inicio del loop
    for i in range(N_arboles):
        regr = DecisionTreeRegressor() # Generamos un modelo de √°rbol de decisi√≥n en cada iteraci√≥n desde cero
        # Muestreo bootstrap desde los √≠ndices del conjunto de entrenamiento, notar que tiene un replace = TRUE lo que permite tener reemplazos en las muestras
        # Este reemplazo hace que se repitan elementos en cada iteraci√≥n generada
        # La funci√≥n que genera esto es de numpy, espec√≠ficamente random.choice
        indices_bagging = np.random.choice(indices, size = len(indices), replace = True)

        # Ajustamos el modelo de arbol para el caso particular con el conjunto de entrenamiento re-muestreado (mediante bootstrapping) para X e Y
        regr.fit(X_train[X_train.index.isin(indices_bagging)],
                 y_train[y_train.index.isin(indices_bagging)])
        # Agregamos el modelo ajustado a la lista de modelos
        objs.append(regr)
        
    return(objs)

La predicci√≥n la realizamos con la agregaci√≥n de todas las predicciones individuales de todos los modelos ajustados en la funci√≥n anterior, la imagen a continuaci√≥n ilustra este proceso (de una forma bastante rara, pero espero se entienda)

def prediccion_arbol(test, model, metric):

    data = [] # Lista vac√≠a para guardar las predicciones
    for i in range(len(model)): # Para cada modelo realizamos la predicci√≥n en el conjunto de testeo
        y_pred = model[i].predict(test)
        data.append(y_pred) # Guardamos las predicciones en la lista generada anteriormente
    data = pd.DataFrame(data) # Manejo de datos para que devuelva una forma tabular
    data = np.transpose(data) # Traspuesta para hacer operaciones row-wise

    # Agregamos dos formas de obtener predicciones
    # Uno a trav√©s del promedio de las filas y otro a trav√©s de la mediana de las predicciones de las filas
    if metric == 'mean':
        y_pred = data.mean(axis=1)
    elif metric == 'median':
        y_pred = data.median(axis=1)
        
    return(y_pred)
